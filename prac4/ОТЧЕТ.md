## 3. Теоретические сведения

### 3.1 Структуризация React-проектов

При разработке крупных React-приложений важно правильно организовать структуру проекта. Основные подходы:
- Разделение на компоненты, модули, страницы
- Паттерн "контейнер-компонент"
- Разделение кода на слои (компоненты, стили, маршруты)

### 3.2 Динамическая маршрутизация

Динамические пути позволяют создавать маршруты с параметрами:
```javascript
<Route path="/dialogs/:userId" component={Dialogs} />
```

Преимущества:
- Гибкость настройки структуры маршрутов
- Автоматическое обновление при изменении параметров
- Возможность создания универсальных компонентов

### 3.3 Использование Props

Props (свойства) - механизм передачи данных от родительского компонента к дочернему:
```javascript
<DialogItem name="Lumine" id="1" />
```

### 4.2 Реализация компонента DialogItem

Создан компонент `DialogItem.js` для отображения элемента диалога:

```javascript
const DialogItem = (props) => {
  let path = "/dialogs/" + props.id;
  
  return (
    <div className="dialog-item">
      <div className="dialog-avatar">
        {props.name.charAt(0).toUpperCase()}
      </div>
      <Link to={path} className="dialog-name">
        {props.name}
      </Link>
    </div>
  );
}
```

**Особенности реализации:**
- Принимает props: `name` (имя) и `id` (идентификатор)
- Формирует динамический путь к диалогу
- Использует Link из react-router-dom для навигации

### 4.3 Реализация компонента Message

Создан компонент `Message.js` для отображения сообщений:

```javascript
const Message = (props) => {
  return (
    <div className={`message ${props.isOwn ? 'message-own' : 'message-other'}`}>
      <div className="message-content">
        <div className="message-author">{props.author}</div>
        <div className="message-text">{props.text}</div>
        <div className="message-time">{props.time}</div>
      </div>
    </div>
  );
}
```

**Принимаемые props:**
- `author` - автор сообщения
- `text` - текст сообщения
- `time` - время отправки
- `isOwn` - флаг, определяющий свое/чужое сообщение

### 4.4 Реализация страницы Dialogs

Создана страница `Dialogs.js`, использующая компоненты:

```javascript
const Dialogs = () => {
  const { userId } = useParams(); // Получение параметра из URL
  
  const currentMessages = userId ? messagesData[userId] || [] : [];
  
  return (
    <div className="dialogs">
      <div className="dialogs-list">
        {dialogsData.map(dialog => (
          <DialogItem 
            key={dialog.id}
            name={dialog.name} 
            id={dialog.id} 
          />
        ))}
      </div>
      
      <div className="messages-container">
        {currentMessages.map(message => (
          <Message
            key={message.id}
            author={message.author}
            text={message.text}
            time={message.time}
            isOwn={message.isOwn}
          />
        ))}
      </div>
    </div>
  );
}
```

**Ключевые моменты:**
- Использование `useParams()` для получения динамического параметра из URL
- Передача данных через props в компоненты DialogItem и Message
- Использование метода `map()` для отображения списков

### 4.5 Настройка маршрутизации

В файле `App.js` настроена маршрутизация:

```javascript
<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/dialogs" element={<Dialogs />} />
  <Route path="/dialogs/:userId" element={<Dialogs />} />
  <Route path="/profile" element={<Profile />} />
</Routes>
```

**Реализованные маршруты:**
- `/` - главная страница
- `/dialogs` - страница диалогов (без выбранного диалога)
- `/dialogs/:userId` - страница диалогов с конкретным пользователем (динамический путь)
- `/profile` - профиль пользователя
