# Инструкция по запуску проекта

## Проблема с выполнением npm в PowerShell

Если при запуске `npm install` возникает ошибка "выполнение сценариев отключено", выполните следующие шаги:

### Вариант 1: Временное разрешение (рекомендуется для учебных целей)
Выполните команду в PowerShell:
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```
После этого запустите:
```powershell
npm install
```

### Вариант 2: Использование CMD
Откройте обычную командную строку (CMD) вместо PowerShell и выполните:
```cmd
npm install
```

### Вариант 3: Использование Git Bash (если установлен)
```bash
npm install
```

## Запуск приложения

После установки зависимостей запустите проект командой:
```bash
npm start
```

Приложение откроется в браузере по адресу: http://localhost:3000

## Структура приложения

После запуска вы увидите:
1. **Главная страница** - описание практической работы
2. **Диалоги** - страница с диалогами и сообщениями
3. **Профиль** - страница профиля пользователя

## Демонстрация работы Props и динамических путей

На странице "Диалоги" можно увидеть:
- Список диалогов (каждый создан через компонент DialogItem с props: name и id)
- При клике на диалог URL меняется на `/dialogs/:userId` (динамический путь)
- Отображаются сообщения для выбранного диалога (через компонент Message с props)

## Основные файлы для проверки

1. **src/components/DialogItem/DialogItem.js** - компонент с использованием props
2. **src/components/Message/Message.js** - компонент сообщения с props
3. **src/pages/Dialogs/Dialogs.js** - использование компонентов и динамических данных
4. **src/App.js** - настройка маршрутизации с динамическим путем
