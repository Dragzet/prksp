# Практическая работа №4
## Оптимизация архитектуры React приложения

### Описание проекта
Данный проект демонстрирует реализацию React-приложения с оптимизированной архитектурой, использующего динамическую маршрутизацию и модульную структуру компонентов.

### Цель работы
Изучение способов улучшения архитектуры React-приложения за счёт создания динамических путей, что направлено на оптимизацию структуры приложения и упрощения процесса навигации пользователя по страницам.

### Выполненные задачи
- ✅ Создана страница диалогов с использованием props
- ✅ Реализован вывод имён и сообщений с использованием "пропсов"
- ✅ Организована модульная структура приложения
- ✅ Настроена динамическая маршрутизация с React Router

### Основные технологии
- **React 18.2.0** - библиотека для создания пользовательских интерфейсов
- **React Router DOM 6.20.0** - библиотека для маршрутизации
- **React Scripts 5.0.1** - инструменты для разработки

### Ключевые особенности реализации

#### 1. Динамическая маршрутизация
Использование параметров URL для динамического отображения контента:
```javascript
<Route path="/dialogs/:userId" element={<Dialogs />} />
```

#### 2. Использование Props
Передача данных между компонентами через props:
```javascript
<DialogItem name="Lumine" id="1" />
<Message author="User" text="Hello" time="10:30" isOwn={true} />
```

#### 3. Модульная структура
Разделение приложения на:
- **Компоненты** (components/) - переиспользуемые элементы UI
- **Страницы** (pages/) - компоненты верхнего уровня
- **Стили** - каждый компонент имеет свой CSS-модуль

### Установка и запуск

1. Установите зависимости:
```bash
npm install
```

2. Запустите приложение в режиме разработки:
```bash
npm start
```

3. Откройте [http://localhost:3000](http://localhost:3000) в браузере

### Функциональность

#### Страница диалогов
- Список диалогов с аватарами
- Динамическое отображение сообщений при выборе диалога
- История переписки для каждого диалога
- Форма для отправки новых сообщений

#### Компоненты
- **DialogItem** - отображает элемент в списке диалогов с аватаром и именем
- **Message** - отображает отдельное сообщение с автором, текстом и временем

### Применённые паттерны

1. **Компонентный подход** - разбиение UI на независимые, переиспользуемые компоненты
2. **Контейнер-компонент** - разделение логики (Dialogs) и презентации (DialogItem, Message)
3. **Динамическая маршрутизация** - использование параметров URL для навигации
