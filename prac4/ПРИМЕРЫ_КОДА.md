## Сравнение статических и динамических путей

### Статические пути:
```javascript
<Routes>
  <Route path="/home" element={<Home />} />
  <Route path="/about" element={<About />} />
  <Route path="/contact" element={<Contact />} />
</Routes>
```

**Использование:**
- `/home` всегда ведёт на страницу Home
- `/about` всегда ведёт на страницу About

### Динамические пути:
```javascript
<Routes>
  <Route path="/dialogs/:userId" element={<Dialogs />} />
  <Route path="/products/:productId" element={<Product />} />
  <Route path="/users/:userId/posts/:postId" element={<Post />} />
</Routes>
```

**Использование:**
- `/dialogs/1`, `/dialogs/2`, `/dialogs/любое_значение` → Dialogs
- `/products/123` → Product (productId = 123)
- `/users/5/posts/42` → Post (userId = 5, postId = 42)

## Ключевые концепции из задания

### 1. Props - передача данных
```javascript
// Родитель передаёт данные
<Component name="значение" id={1} />

// Ребёнок получает через props
const Component = (props) => {
  return <div>{props.name}</div>;
}
```

### 2. Динамические пути
```javascript
// Определение маршрута с параметром
<Route path="/page/:param" element={<Page />} />

// Получение параметра в компоненте
const { param } = useParams();
```

### 3. Map для создания списков
```javascript
const items = [
  { id: 1, name: "Item 1" },
  { id: 2, name: "Item 2" }
];

return (
  <div>
    {items.map(item => (
      <Item key={item.id} name={item.name} />
    ))}
  </div>
);
```

### 4. Модульная структура
- **components/** - переиспользуемые компоненты
- **pages/** - компоненты-страницы
- Каждый компонент в отдельной папке с CSS
